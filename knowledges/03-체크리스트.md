# 03 · 새 LLM 생성 기능 만들 때 체크리스트

[01](01-llm-환각-잡기.md)·[02](02-프롬프트-레시피.md)를 매번 다시 읽지 않아도 되게, 그대로 따라갈 점검표.

## 프롬프트 설계
- [ ] **역할 한 줄** 부여했나? ("당신은 ○○ 검증 편집자입니다")
- [ ] 재작성·검증이면 **`원본:`을 항목마다 동봉**했나? (그라운딩 — 환각의 1차 방어선)
- [ ] "원본에 없는 사실 추가·변경 금지(왜곡 금지)"를 **명시**했나?
- [ ] 항목 간 흐름이 필요한 작업이면 **독립 완결 문장 규칙(`_INDEP`)** 을 넣었나?
- [ ] 출력이 단조롭지 않게 **다양성 규칙**(어미·표현 번갈아, 연속 금지)을 넣었나?
- [ ] **"반드시 이 형식 그대로(다른 말 금지)"** + 한 항목=한 줄 형식을 못박았나?
- [ ] 길이 제한(자막 22~40자 등)과 금지(명사 끝맺음·따옴표·설명) 명시했나?
- [ ] 긴 입력은 **`[:N]` 컷**으로 토큰 폭주를 막았나?

## 워크플로 설계
- [ ] **검증과 수정을 분리**했나? (검증기는 원문 불변 + 대안만, 수정기는 따로)
- [ ] 개별 패치 루프 대신 **전체 1회 재작성** 경로가 있나? (수렴)
- [ ] 재작성 직후 **자동 사실검증 1회**를 연결했나?
- [ ] 최종 적용 전에 **사람이 확인/클릭**하는 관문이 있나?

## 파싱·안정성
- [ ] 형식에 맞는 줄만 취하는 **정규식 파서**가 있나? (전각 콜론·파이프 `[:：]`, `[|｜]` 허용)
- [ ] 구분자가 모델이 흉내 못 낼 만큼 특이한가? (`;;`, `|`)
- [ ] CLI subprocess 경유면 **제어문자 제거(`_CTRL_RE`)** 를 거쳤나?
- [ ] LLM 미설치/미로그인 시 **조용히 비활성**되고 핵심 기능(렌더 등)은 도나? (`available()` 가드)

## 검증(직접 돌려보기)
- [ ] 생성물에 씬 간 의존 접속어('이에 맞서/뒤이어/그')가 사라졌나?
- [ ] **두더지잡기 재현 케이스**로 전체 재작성 → 다시 돌려도 새 지적이 거의 안 나오나(수렴)?
- [ ] **알려진 사실 왜곡 케이스**가 그라운딩+자동검증으로 고쳐지거나 빨간 박스로 잡히나?
- [ ] 형식 깨짐 없이 전 항목이 파싱되나?
