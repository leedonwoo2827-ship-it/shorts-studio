# knowledges — LLM 콘텐츠 생성 노하우

> 쇼츠공방을 만들며 잡은, **"요약·재생성할 때 끝없이 나오는 환각/표류"** 를 길들인 방법과
> 실제로 쓴 프롬프트를 모아둔 곳. 다른 앱(문서공방·영상공방 등)에도 그대로 가져다 쓰려고 정리한다.

---

## 🕸️ 이 레포의 핵심 = **5단계 방어망**
> 환각·표류를 한 방에 막는 마법은 없다. **단계를 겹쳐(그물을 여러 겹 쳐서)** 앞 단계가 놓친 걸
> 뒤 단계가 잡는 구조가 핵심이다. **단계 수와 순서가 곧 안정성**이다.

```
입력(원본)
  │
  ├─ 1단계 · 독립 완결 문장   씬 간 의존 접속어 금지     → 비수렴(두더지잡기) 차단
  ├─ 2단계 · 그라운딩          매 항목에 `원본:` 동봉      → 환각(사실 왜곡) 차단
  ├─ 3단계 · 전체 1회 재작성   개별 패치 루프 폐기         → 국소 패치 연쇄 불안정 제거(수렴)
  ├─ 4단계 · 자동 사실검증     재작성 직후 자동 1회        → 재작성이 남긴 잔여 오류 포착
  └─ 5단계 · 사람 최종 관문    대안 클릭으로 사람이 적용   → 자동화하되 사람이 마지막 게이트
  ▼
출력(안정)
```
| 단계 | 처방 | 막는 실패 |
|:--:|---|---|
| **1** | 독립 완결 문장(`_INDEP`) | 수렴 실패 |
| **2** | 그라운딩(`원본:` 동봉) | 사실 왜곡 |
| **3** | 전체 1회 재작성(`tidy_all`) | 국소 패치 연쇄 불안정 |
| **4** | 자동 사실검증(`verify_captions`) | 잔여 오류 |
| **5** | 사람 최종 확인 | 자동화 과신 |

(파싱 안정·CLI 잡티 제거는 모든 단계를 떠받치는 **기반 처리**. 자세히는 [01](01-llm-환각-잡기.md) 패턴 5·6.)

## 무엇이 문제였나
LLM에게 "자막을 다듬어줘 / 흐름을 검토해줘"를 반복하면 두 가지가 계속 터졌다.

1. **사실 왜곡(환각)** — 원본에 없던 사실·관계를 지어냄.
   예: "명나라와 청나라는 조공을 *주고받으며*…" (실제론 주변국이 명·청에 조공).
2. **수렴하지 않음(두더지잡기)** — 한 씬을 고치면 이웃 씬이 어색해지고, 다시 고치면 또 다른 곳이
   어색해져 **무한 반복**. 원인은 "이에 맞서·뒤이어·그래서" 같은 **씬 간 의존 접속어**.
