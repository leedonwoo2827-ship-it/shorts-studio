# knowledges — LLM 콘텐츠 생성 노하우

> 쇼츠공방을 만들며 잡은, **"요약·재생성할 때 끝없이 나오는 환각/표류"** 를 길들인 방법과
> 실제로 쓴 프롬프트를 모아둔 곳. 다른 앱(문서공방·영상공방 등)에도 그대로 가져다 쓰려고 정리한다.

---

## 🕸️ 이 레포의 핵심 = **5단계 방어망**
> 환각·표류를 한 방에 막는 마법은 없다. **단계를 겹쳐(그물을 여러 겹 쳐서)** 앞 단계가 놓친 걸
> 뒤 단계가 잡는 구조가 핵심이다. **단계 수와 순서가 곧 안정성**이다.

```
입력(원본)
  │
  ├─ 1단계 · 독립 완결 문장   생성·검증 공통 규칙(씬 간 접속어 금지)  → 비수렴(두더지잡기) 차단
  ├─ 2단계 · 그라운딩          매 항목 `원본:` 대조                    → 환각(사실 왜곡) 차단
  ├─ 3단계 · 자동 전체 검증    구성 직후 1회 자동 사실검증            → 어색·왜곡 1차 적발
  ├─ 4단계 · 씬별 검토         사람이 의심 씬만 🔎로 정밀 재검         → 잔여 어색·오류 적발
  └─ 5단계 · 대안 분리 + 사람 적용  AI는 원문 불변·대안만, 사람이 클릭  → 손수정 보존 & 최종 게이트
  ▼
출력(안정)
```
| 단계 | 처방 | 막는 실패 |
|:--:|---|---|
| **1** | 독립 완결 문장(`_INDEP`) | 수렴 실패 |
| **2** | 그라운딩(`원본:` 대조) | 사실 왜곡 |
| **3** | 자동 전체 검증(`verify_captions`) | 어색·왜곡 누락 |
| **4** | 씬별 정밀 검토 | 잔여 오류 |
| **5** | 대안 분리 + 사람 적용 | 자동화 과신·손수정 유실 |

> ⚠️ **버린 길 — "전체 1회 재작성"(`tidy_all`):** 모든 자막을 한 번에 다시 쓰면 수렴은 빠르지만,
> **사람이 손으로 고친 것까지 전부 덮어쓴다.** 그래서 이 앱은 빼고, 검증은 하되 **고치는 건 대안을
> 사람이 골라 적용**(3~5단계)하는 길로 단일화했다. 코드(`tidy_all`/`/api/tidy`)는 *손수정이 없는*
> 대량 초안 정리용으로 남겨둠 — **사람이 다듬기 시작했다면 쓰지 말 것.** ([01](01-llm-환각-잡기.md) 패턴 3·4)

(파싱 안정·CLI 잡티 제거는 모든 단계를 떠받치는 **기반 처리**. 자세히는 [01](01-llm-환각-잡기.md) 패턴 5·6.)

## 무엇이 문제였나
LLM에게 "자막을 다듬어줘 / 흐름을 검토해줘"를 반복하면 두 가지가 계속 터졌다.

1. **사실 왜곡(환각)** — 원본에 없던 사실·관계를 지어냄.
   예: "명나라와 청나라는 조공을 *주고받으며*…" (실제론 주변국이 명·청에 조공).
2. **수렴하지 않음(두더지잡기)** — 한 씬을 고치면 이웃 씬이 어색해지고, 다시 고치면 또 다른 곳이
   어색해져 **무한 반복**. 원인은 "이에 맞서·뒤이어·그래서" 같은 **씬 간 의존 접속어**.
